# Architecture documentation

- [overview.md](overview.md) — top-level layout + layer responsibilities + invariants
- [layers.md](layers.md) — the allowed-import matrix
- [adr/](adr/) — Architecture Decision Records for non-obvious calls

ADRs are short, dated, and explain **why** something is the way it is — not what it does. Read the ADR when you're about to argue with a structural choice; either you'll agree, or the ADR will tell you when to revisit it.
