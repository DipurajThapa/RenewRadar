# Renewal Radar

A SaaS subscription and notice-deadline manager for IT/Ops Leads at 25–500 person companies. Built per the FSD in [`../v3/`](../v3/).

## Quick start

```bash
# 1. Install dependencies
pnpm install

# 2. Configure environment
cp .env.example .env.local
# Then fill in real values — see .env.example comments for where to get each

# 3. Push the schema to your Neon Postgres
pnpm db:push

# 4. (Optional) Seed local dev data
pnpm db:seed

# 5. Run the dev server
pnpm dev
# Open http://localhost:3000
```

## Where things live

- **Routes:** `src/app/` — Next.js 14 App Router
- **Components:** `src/components/` — domain-grouped, plus `src/components/ui/` for shadcn primitives
- **Server logic:** `src/lib/` — `db/` (Drizzle), `auth/`, `billing/`, `email/`, `notice-deadline/`, `validation/`
- **Scheduled jobs:** `src/inngest/`
- **Email templates:** `src/emails/`
- **DB migrations:** `drizzle/` (auto-generated by `pnpm db:generate`)

## Architecture decisions

See [`../v3/starter/README.md`](../v3/starter/README.md) for the full rationale. Short version:

- **Next.js 14 App Router** — single repo, frontend + API + Server Actions together
- **Drizzle ORM + Neon Postgres** — type-safe queries, branching for dev/prod
- **Clerk auth** — fastest path to working auth at solo-founder scale
- **Server Actions for all mutations** — typed end-to-end
- **Server Components by default** — less JS shipped, faster pages
- **Zod for validation** — pairs cleanly with Server Actions
- **shadcn/ui + Tailwind** — copy-paste components, no dependency hell
- **Stripe Checkout + Customer Portal** — never built a billing UI
- **Inngest for cron + delayed jobs** — daily notice-deadline alerts run here
- **Resend + React Email** — transactional email
- **Sentry** — error tracking
- **TypeScript strict mode** — `noUncheckedIndexedAccess` is on; the compiler is your second pair of eyes

## Setting up the third-party services

You need accounts for:

1. **Neon** (neon.tech) — get the pooled connection string → `DATABASE_URL`
2. **Clerk** (clerk.com) — get publishable + secret keys; create webhook endpoint pointing to `<your-url>/api/webhooks/clerk`
3. **Resend** (resend.com) — verify your sending domain; get API key
4. **Stripe** (stripe.com) — create Starter/Growth/Pro products; get test-mode keys; set up webhook to `<your-url>/api/webhooks/stripe`
5. **Inngest** (inngest.com) — register your app's `/api/inngest` endpoint
6. **Sentry** (sentry.io) — optional in dev; required for production
7. **Vercel** (vercel.com) — for hosting; auto-deploys from GitHub

See [`../v3/starter/00_Setup_Day_1.md`](../v3/starter/00_Setup_Day_1.md) for the full account-setup walkthrough.

## The five binding principles

These are baked into the code and copy. Don't remove them:

1. **Advisor, never agent** — drafts only, customer sends every external communication
2. **No money movement** — no bank/card feeds, no vendor invoice payment
3. **Self-serve onboarding under 30 minutes** — no implementation services
4. **Manual-first, integration-later** — V1 has no vendor APIs
5. **Privacy by default** — never pool, share, or sell customer data

If you're tempted to remove any of these, re-read [`../v3/00_FSD_Overview.md`](../v3/00_FSD_Overview.md).
